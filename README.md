## 1. 기본 소개

### 한 줄 요약
알람을 끄려면 **전면 카메라로 타임랩스 촬영을 시작**해야 한다. 그냥 끌 수 없기 때문에, 예약한 그 시각에 실제로 그 자리에 앉게 만든다.

### 핵심 루프
```
다짐(예약) → 알람 → 거치 가이드 → 촬영(세션) → 완주/실패 판정
        → 상점·벌점 원장 기록 → 캘린더·연속 달성일 → 활동 슬롯 확장
```

### 특징 한눈에
| 요소 | 설명 |
|---|---|
| **다짐(활동) 예약** | 시간·요일·지속시간을 정해 반복/일회성으로 예약 (겹침 금지, 시작 30분 전 편집 잠금) |
| **알람 해제 = 촬영 시작** | 알람은 촬영을 시작해야만 꺼진다 (10분 내 미시작 시 노쇼) |
| **타임랩스 기록** | 전면 카메라로 세션 전체를 타임랩스로 압축 저장 (기기 내에만) |
| **상점·벌점** | 완주·실패·노쇼·자리비움에 따라 점수가 쌓이는 원장 |
| **연속 달성일 → 슬롯 확장** | 꾸준히 성공하면 동시에 걸 수 있는 활동 수가 늘어난다 |
| **강도 선택** | 매운맛(유예 있음) / 미친 매운맛(유예 없음, 점수 2배 — 완주 3회 후 잠금 해제) |
| **멤버십** | 월 4,400원 구독 — 슬롯·워터마크·미친 매운맛 즉시 해제 혜택 |

### 로그인 방식
이메일 / Google / Apple 로그인 + **게스트 모드**. 상점·벌점은 계정 단위로 저장됩니다. 이메일 가입 시 인증이 필요하고, 만 14세 이상만 이용할 수 있습니다.

## 2. 운영자용 — 상세 정책

> 모든 수치는 소스 코드(`Models.swift`, `SessionEngine.swift`, `TimeLockApp.swift`, `CameraRecorder.swift`)의 실제 값과 일치합니다. 정책을 바꾸려면 **여기 + iOS 상수 + 안드로이드(`android/.../models/Models.kt`)를 반드시 함께** 수정하세요 — 이 문서가 양 플랫폼의 단일 기준입니다.

### 2.1 시간 정책 (`TimePolicy`)
| 항목 | 값 | 초과 시 |
|---|---|---|
| 알람 후 촬영 시작 창 | **10분** (600초) | 노쇼 탈락 |
| 긴급 중단 후 재촬영 창 | **10분** (600초, 세션당 누적 예산) | 이탈 실패 |
| 활동 지속시간 옵션 | 10·15·25·30·45·60·90·120·150·180·240·300·360·480분 | — |
| 기본 예약 시작 시각 | 현재시각 + 2시간을 정각으로 반올림 (예: 9:39 → 11:00) | — |

### 2.2 강도 (`Intensity`) — 앱 전역 단일 값
| 강도 | 긴급 중단 | 점수 배율 | 설명 |
|---|---|---|---|
| 🌶️ **매운맛** (spicy) | 허용 (10분 내 재촬영 시 벌점 없음) | 1배 | 기본 |
| 🔥 **미친 매운맛** (insane) | 없음 (이탈 즉시 실패) | **상점 2배 · 벌점 2배** | 하이 리스크 하이 리턴 |

- **잠금 해제**: 미친 매운맛은 **매운맛 완주 3회** 후 해제됩니다 (온보딩·강도 설정 화면에 진행 상황 `n/3` 표시). **멤버십 회원은 조건 없이 즉시 사용 가능.** 해제되는 완주 순간(완주 3회 달성 시) **잠금 해제 보너스 +5 상점**이 파티클과 함께 지급됩니다 (계정당 평생 1회).
- **변경 규칙**: 강도를 **올리는 건 즉시 적용**, **내리는 건 다음날 0시부터 적용** (당일 회피 방지).

### 2.3 점수 규칙 (`ScoreRules`)
`배율 = 미친 매운맛이면 2, 아니면 1`

**완주 상점은 활동 길이에 따라 커집니다** (벌점은 길이와 무관하게 고정):

| 활동 길이 | 완주 상점 |
|---|---|
| 10분 ~ 1시간 | **+10** × 배율 |
| 1시간 30분 ~ 3시간 | **+20** × 배율 |
| 4시간 ~ 8시간 | **+30** × 배율 |

시간 설정 UI(예약 편집·즉시 시작) 옆에 **"완료 시 +n점" 태그**로 미리 표시됩니다.

| 결과 | 이벤트 | 점수 |
|---|---|---|
| 완주 | 완주 상점 | **+10/+20/+30** × 배율 (길이별, 위 표) |
| 이탈 실패 | 이탈 벌점 | **−10** × 배율 |
| 노쇼 (10분 미시작) | 노쇼 벌점 | **−15** × 배율 |
| 긴급 종료 (세션 포기) | 긴급 벌점 | **−5** × 배율 |
| 안전 종료 (배터리·저장공간·카메라 실패) | — | **0** (벌점 없음) |
| 슬롯 확장 순간 | 슬롯 확장 보너스 | **+5** |
| 미친 매운맛 잠금 해제 순간 (매운맛 완주 3회째) | 잠금 해제 보너스 | **+5** (계정당 평생 1회) |

**세션 결과 종류**: 완주(completed) / 이탈 실패(exitFailed) / 노쇼(noShow) / 긴급 종료(emergency) / 안전 종료(safetyEnded — 캘린더 색 판정에서 중립)

### 2.4 완주 판정과 세션 중 이벤트
- **완주 판정**: "**순수 촬영 시간**"(틱 1초 단위)이 목표 시간에 도달하면 자동 완주 종료. 긴급 중단 동안은 틱이 멈추므로 그 시간만큼 종료 시각이 자연히 밀립니다.
- **이탈 이벤트** = 앱 백그라운드 전환 · 화면 잠금(전원 버튼) · 앱 강제 종료:
  - 매운맛: 긴급 중단과 동일 취급 — 자동으로 브레이크 시작 (예산 차감)
  - 미친 매운맛: **즉시 실패 확정** (되돌릴 수 없음)
- **통화 수신**: 특별취급 없음 — 전화가 오면 앱이 백그라운드로 내려가 **이탈과 동일**하게 처리됩니다(매운맛 긴급용무 수동 재개 / 미친맛 즉시 실패). "앵그리모드 미사용은 세션당 10분까지만" 원칙에 따름. (iOS·안드로이드 통일, 전화 상태 권한 불필요)
- **안전 종료** (벌점 없음, 30초마다 점검):
  - 배터리 **5% 이하** + 충전 중 아님
  - 저장 공간 **500MB 미만**
  - 카메라 시작 실패
- **긴급 종료** (세션 포기): 사유를 입력받아 기록하고 벌점 −5×배율.

### 2.5 자리비움 감지 (Vision, 온디바이스)
- 촬영 프레임을 **5초 간격**으로 검사 (`VNDetectHumanRectanglesRequest`, 완전 온디바이스 — 네트워크·외부 모델 없음)
- **판정 기준은 '얼굴'이 아니라 '사람의 몸'** — 뒷모습·고개 숙임도 몸 포즈 감지가 잡는다.
  iOS: 상반신 사각형 ∨ 몸 포즈(관절 4개↑) / 안드로이드: 얼굴 ∨ 몸 포즈(관절 4개↑).
  **움직임은 재석 근거가 아니다** — 흔들리는 물건이 빈자리를 재석으로 못 만들고,
  미동 없이 몰입한 사람은 몸이 보이는 한 재석이다.
- 연속 **30초** 부재 → 경고 배너 + **카운트 +1**. 카운트는 **2분 안에 복귀해도 되돌아가지 않고 세션 내내 누적**되며, 배너에 `n/3`으로 표시됨. 3번째 경고는 "마지막 경고 — 다음 자리비움은 알림 없이 자동 중단" 안내.
- 다음 두 조건 중 하나면 부재 확정 — "경고는 3번까지만 봐준다" (부재 자체엔 벌점 없음):
  1. 경고 후 연속 **120초(2분)** 부재
  2. **4번째 자리비움 발생** (경고 없이 감지 30초 만에 즉시 — 복귀 여부 무관)
- 부재 확정 시:
  - **매운맛**: **자동 긴급 중단** (긴급 용무 버튼을 누른 것과 동일 — 남은 예산만큼 10분 재촬영 창이 열림. 안 돌아오면 창 만료로 실패 = 이탈 벌점 −10×배율. 반복될수록 예산이 닳아 결국 실패로 수렴)
  - **미친 매운맛**: 긴급 용무 기능이 없으므로 **즉시 실패** (이탈 벌점)
  - 벌점은 오직 **실패가 확정될 때만** 이탈 벌점으로 부과 — 자리비움 전용 벌점은 없음

**왜 예산(10분)과 경고 횟수(3회) 두 장치가 모두 필요한가:**

| 장치 | 측정 대상 | 잡는 패턴 |
|---|---|---|
| 10분 긴급 예산 | 자리를 비운 **총 시간** | 한 번 나가서 오래 안 돌아옴 |
| 경고 3회 한도 | 자리를 비운 **횟수** | 짧게 여러 번 들락날락 |

30초~2분 사이에 복귀하기를 반복하면 예산은 전혀 닳지 않는다 — 경고 카운트가 없으면 이 패턴으로 자리에 없는 채 완주하는 구멍이 생긴다. 경고는 3번까지만 봐주고 4번째 부재부터는 알림 없이 곧바로 중단되므로 "잠깐씩 자주 비우기"가 차단되고, 그 뒤로는 중단→복귀마다 예산이 차감되어 결국 실패로 수렴한다.

### 2.6 긴급 중단(브레이크) 예산
- 매운맛 전용 기능 (미친 매운맛은 브레이크 없음)
- 세션당 총 **10분**, 브레이크를 쓸 때마다 차감되며 **리셋되지 않고 누적**됨
  - 예: 1분 쓰고 재촬영 → 다음 브레이크는 9분부터 시작
- 남은 예산이 **0이 되면 즉시 실패 종료** (버튼을 다시 누를 필요 없음)
- 예산 소진 시 긴급 버튼은 **비활성('긴급 소진')**

### 2.7 활동 슬롯 정책 (`SlotPolicy`) — "현재 연속 달성일" 기준
슬롯(동시에 걸 수 있는 활동 수)은 **항상 현재 연속 달성일**이 정합니다. 연속이 오르내리면 한도가 자동으로 따라갑니다.

| 연속 달성일 | 최대 활동 수 |
|---|---|
| 기본 (0~2일) | 2개 |
| **3일** | 3개 |
| **5일** | 4개 |
| **7일** | 5개 |
| **10일** | 10개 |
| **30일** | 무제한 |

- **자동 하향**: 연속이 끊기면 한도가 내려가지만, **이미 만든 예약은 삭제되지 않고** 새 활동 추가만 제한됩니다. (업/다운그레이드 별도 로직 불필요)
- **멤버십 회원**: 연속일과 무관하게 **최소 10개 보장** (30일 무제한은 동일 적용)
- **슬롯 확장 보너스**: 연속일이 슬롯 확장 단계(3·5·7·10·30일)를 넘는 순간 완주하면 **+5 상점 + 파티클 효과** (계정별 최고 도달 단계를 기록해 중복 지급 방지, 연속이 깨져 단계 아래로 내려가면 재획득 가능)

**연속 달성일 정의**: 오늘(기록 없으면 어제)부터 거꾸로, **실패 없이 완주한 날**을 연속으로 카운트. (캘린더 대시보드와 동일 정의)

### 2.8 예약·알람·촬영 준비 규칙
- **예약 생성**: 활동명 필수, 같은 날 **시간 겹침 금지**, 슬롯 한도 내에서만 신규 생성. 반복(요일 선택) 또는 일회성.
- **노쇼 판정 범위**: 예약 **생성 시각 이후의 발생분만** 노쇼 대상. 오후에 "매일 오전 6시" 반복을 만들어도 이미 지난 오늘/어제 6시가 소급 노쇼로 찍히지 않음.
- **편집 잠금**: 다음 시작 **30분 전부터 편집 불가** (직전 회피 방지).
- **태그 프리셋**: 공부 · 독서 · 운동 · 작업 · 악기 · 글쓰기
- **즉시 시작 ("지금 바로 시작")**: 활동명을 입력해야만 촬영 준비로 진행 가능.
- **알람 화면**: [촬영 준비] / [일정 취소] 슬라이드 — 취소 시 **사유를 입력받아 기록**.
- **거치 가이드**: 촬영 **방향(세로/가로)을 선택하면 촬영 내내 고정** (UI 회전 잠금). 3·2·1 카운트다운 후 촬영 시작, 실수 방지용 [취소하기] 버튼 제공 (카운트다운 시작 후엔 비활성).

### 2.9 촬영본(타임랩스) 정책
- 전면 카메라, 재생 **30fps**, 목표 길이에 맞춰 캡처 간격을 동적으로 계산해 세션 전체를 타임랩스로 압축
- **기기 내에만 저장** (`Documents/Sessions/`, iOS 완전 암호화 파일 보호). **서버 전송 없음**
- 세션 종료 화면에서 **저장하지 않으면 자동 즉시 삭제**
- 워터마크 **`AngryMoti`** (멤버십 회원은 제거)

### 2.10 멤버십 (유료 구독)
- **월 4,400원**, 자동 갱신 (StoreKit 2, 상품 ID `com.timelock.pro.monthly`)
- 혜택:
  1. 활동 슬롯 최소 10개부터 시작 (무료는 2개)
  2. 타임랩스 워터마크 제거
  3. 미친 매운맛 즉시 잠금 해제 (완주 3회 조건 면제)
  4. **그룹 챌린지** — 초대코드로 모여 같은 일정으로 랭킹 대결 (멤버십 전용. 비구독자는 그룹 탭이 잠금 상태로 표시되며, 구독 만료 후에도 이미 참여 중인 방의 열람·관리는 유지)
  5. 그 외 추가되는 멤버십 기능 모두 포함
- 결제·환불은 Apple App Store를 통해 처리 (회사가 직접 환불 불가)

### 2.11 그룹 챌린지 (`GroupPolicy` / `GroupStore`)
초대코드로 모인 계정들이 **동일한 일정으로 대결**하는 시스템. 하단 토글의 세 번째 탭(👥 그룹).

**방 생성 (방장)**
- 방장이 정함: 방 이름(=참여자 전원의 활동명) · 시작 시각 · 활동 길이 · 반복 요일 · 강도(매운맛/미친매운맛) · 기간 · 방장 닉네임
- 기간: 시작일은 **최소 내일부터**, 종료일은 시작일로부터 **최대 3개월**
- 생성 시 **초대코드(영문 대문자+숫자 5자리, 0/O/1/I 제외)** 발급 — **방장에게만 표시**
- 방장은 시작 전 해체 가능(참여자에게 안내 표시). 시작 후엔 해체 불가, 특권 없음

**참여**
- 초대코드 + 닉네임(방 안 전용, 중복 불가·선착순) 입력. **시작 전까지만**, 최대 **30명**
- 입장 시 기존 개인 예약과 **일정 겹침 검사** — 겹치면 입장 차단
- **그룹도 활동 슬롯 1개를 차지한다** — 슬롯이 가득 차면 생성·참여 모두 차단 (슬롯 확장 동기 유지)
- 미친매운맛 방은 미해제 유저도 입장 가능, **그 그룹 일정에서만** 미친매운맛 적용
- 여러 방 동시 참여 가능(겹침 검사만 통과하면). **게스트는 그룹 탭 자체가 안 보임**

**시작·진행**
- 그룹 예약은 **참여(생성) 시점에 미리 만들어지되 시작일 전에는 발생·알람이 없다** —
  덕분에 시작 시각 정각의 첫 알람이 앱을 열지 않아도 울린다 (👥 표시, **수정·삭제 완전 잠금**)
- 시작 시각에 **2명 미만**이면 방 자동 삭제 + 방장에게 안내, 예약·잘못 찍힌 노쇼 기록까지 자동 정리
- 노쇼는 **그룹 시작일 기준**으로 집계 (앱을 늦게 열어도 소급 계산 — 잠수 이득 차단)
- 그룹 점수 = **그 그룹 일정에서 나온 상벌점만** 0점부터 집계 (슬롯·해제 보너스 제외).
  같은 점수는 개인 누적에도 그대로 가산
- 랭킹: 7명 이하 전체 표시 / 초과 시 1~5등 + "⋯" + 내 등수. **동점은 공동 등수**

**탈퇴·종료**
- 시작 전: 자유 탈퇴 / 시작 후: **벌점 -50**(그룹+개인 모두, 강도 무관 고정) + 남은 일정 삭제, 랭킹에 "중도 포기" 표시
- 종료 후: 최종 결과표(등수/닉네임/총점)만 열람 → **30일 뒤 서버 자동 삭제**. 본인이 '방 나가기' 하면 즉시 목록에서 제거(재조회 불가)
- 판정은 각자 기기에서 수행 후 점수만 서버 합산 — 지인 신뢰 기반 대결 (v1 수용)

**서버 구조 (Firestore)**: `groups/{roomID}` 방 설정 + `groups/{roomID}/members/{uid}` 닉네임·점수 + `users/{uid}.groupIDs` 참여 목록. 수명 주기는 서버 함수 없이 클라이언트가 게으르게 처리.

**플랫폼**: iOS(`GroupStore.swift`)·안드로이드(`GroupStore.kt`) 모두 구현 — 같은 Firestore 컬렉션을 공유하므로 **iOS와 안드로이드 사용자가 한 방에서 대결 가능**. 정책·해체 정리 로직 1:1 동일.

### 2.12 계정·법무
- 로그인: 이메일 / Google / Apple / 게스트. 이메일 가입 시 인증 필수, 만 14세 이상
- **크로스 기기 동기화** (`AccountStore.syncFromCloud` — 앱 시작·복귀·로그인 시 실행):
  - **점수 원장**: 발생 시 업로드(mirror) + 다운로드 병합 (이벤트 UUID 기준 중복 제거)
  - **개인 예약**: `users/{uid}/reservations` 양방향 병합 — `updatedAt` 최신 쪽이 승리,
    삭제는 소프트 삭제(isActive=false)로 전파. 그룹 예약은 GroupStore가 방 문서에서 재생성하므로 제외
  - **노쇼 이중 벌점 방지**: 노쇼 이벤트 ID = MD5("noshow|예약ID|발생시각") — 두 기기가 같은
    노쇼를 각자 스윕해도 클라우드에서 문서 1개로 합쳐짐 (양 플랫폼 동일 해시)
  - **멤버십**: 구독 활성 시 `users/{uid}.proExpiresAt` 기록(애플=실제 만료일, 구글=35일 추정 슬라이딩)
    → 반대 플랫폼은 Pro 판정 = 스토어 구독 ∨ 클라우드 유효. 한쪽 구독으로 양쪽 잠금 해제
  - **홈 다짐(목표) 문구**: `users/{uid}.homeGoal` + `homeGoalUpdatedAt` — 최신 쪽이 승리
  - 세션(캘린더) 기록은 기기별 보관 (영상이 기기에 있으므로)
- **안드로이드 뒤로가기**: 홈 셸에 없던 계층형 back 처리를 추가 — 하위 화면(캘린더·마이페이지·예약편집·
  그룹 탭 내부 화면)에서는 홈으로 복귀, 홈 화면(어느 하단 탭이든)에서는 시스템 기본 동작(배경으로) 유지
- 계정 삭제: **마이페이지 → 프로필 및 구독 관리 → 계정 삭제** — 기기·서버 데이터 즉시 완전 삭제
- 법적 문서: [이용약관](docs/이용약관.md), [개인정보처리방침](docs/개인정보처리방침.md) (노션 공개 페이지, 앱 내 링크는 `Legal.swift`)
- 개인정보보호 책임자: 윤세현 (대표) / 담당자 박진수 (개발자)

## 3. 개발자용 — 소스와 구조

### 3.1 기술 스택
| 영역 | 사용 |
|---|---|
| UI | SwiftUI (iOS 17.0+) |
| 영속성 | SwiftData (`@Model` / `@Query` / `ModelContext`) |
| 카메라 | AVFoundation (`AVCaptureSession`, `AVAssetWriter`, `RotationCoordinator`) |
| 부재 감지 | Vision (`VNDetectHumanRectanglesRequest`) — **완전 온디바이스** |
| 통화 처리 | 특별 감지 없음 — 전화는 백그라운드 이탈로 처리(매운맛 긴급용무·미친맛 즉시 실패) |
| 알림 | UserNotifications (로컬 알림 — 알람/재촬영/실패 안내) |
| 구독 | StoreKit 2 (자동 갱신, `.storekit` 테스트 구성 포함) |
| 백엔드/인증 | Firebase (Auth + Firestore) + GoogleSignIn |

> Firebase·GoogleSignIn은 **`#if canImport(...)`로 격리**되어 있어, 패키지가 없어도 빌드됩니다. 미연동 시 '기기 내 계정' 오프라인 폴백으로 동작(개발/시뮬레이터용). Firebase 발송 메일은 `Auth.auth().languageCode = "ko"`로 한국어 템플릿.

**외부 ML/네트워크 의존 없음** — 자리비움 감지는 Apple Vision 프레임워크만 사용. CI는 GitHub Actions에서 Xcode 빌드 검증.

### 3.2 앱 흐름 (상태 머신)
`RootView`가 3단계를 분기: **온보딩 → 인증(로그인/게스트) → 메인**

촬영 플로우는 `AppState.route` 상태 머신으로 관리되며, **하나의 `fullScreenCover` 안에서 내용만 즉시 교체**합니다 (화면당 커버를 새로 올리면 전환마다 홈이 몇 초씩 노출되는 갭이 생기기 때문).

```
route: none → alarm → mountGuide → session → result
                    (거치 가이드)   (촬영)    (결과)
```

메인 화면(`HomeShellView`)은 **활동 | 일정** 토글 셸이며, 설정은 우상단 마이페이지로 이동.

**세션 중 이벤트 소스**: ScenePhase(백그라운드, 통화 포함) · protectedData(화면 잠금) · 배터리/저장공간 폴링(30초) · Vision 부재 폴링(5초). 각 이벤트의 정책은 §2.4 참고.

### 3.3 데이터 모델 (SwiftData)
모든 엔티티는 `ownerUserID`로 계정별 격리 (게스트는 `"guest"`).

| 모델 | 역할 |
|---|---|
| `Reservation` | 다짐(활동 예약) — 시작 분·지속시간·반복 요일/일회성 날짜, 겹침 판정(`overlaps`) |
| `FocusSession` | 수행 기록 — 목표/실제 촬영 시간, 결과, 긴급 사유, 영상·썸네일 파일명 |
| `ScoreEvent` | 점수 원장 트랜잭션 — 타입·점수·세션ID·강도·시각 |

**점수 원장 불변 설계**: 규칙이 바뀌어도 재계산이 가능하도록 원본 이벤트를 그대로 보존. 보너스·보정도 기존 레코드를 삭제하지 않고 **상쇄/추가 이벤트로만** 기록합니다.

### 3.6 주요 설계 결정 (그리고 그 이유)
- **프리뷰 레이어 재부모화**: `AVCaptureVideoPreviewLayer.session` 재연결이 실행 중인 세션에서 메인 스레드를 수 초간 블록 → 거치 가이드→세션 전환마다 "시작!" 후 화면 멈춤. 프리뷰 뷰를 단일 인스턴스로 재부모화해 재연결을 1회로 제한하여 해결.
- **틱 기반 타이머**: 표시 시간을 프레임 수 × 동적 캡처 간격으로 계산하니 5초씩 점프 → 벽시계 틱(1초)으로 전환. 중단 중엔 틱이 멈춰 '순수 촬영 시간'과 항상 일치.
- **슬롯 = 현재 연속일**: 업그레이드/다운그레이드를 별도 상태로 두지 않고 "현재 연속일이 곧 한도"로 정의해 엣지 케이스 제거.
- **불변 원장 + 상쇄 이벤트**: 리셋·보너스·계정 삭제 보정을 모두 새 이벤트로만 처리 → 히스토리 훼손 없이 재계산 가능.
- **촬영 방향 사전 잠금**: 구도 단계에서 세로/가로를 선택하고 세션 내내 단일 방향 마스크로 고정 → 촬영 중 기기를 돌려도 UI·영상이 요동치지 않음.
