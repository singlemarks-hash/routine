# 앵그리모티 — 안드로이드(Google Play) 배포 가이드

> 실제 출시 때 위에서부터 순서대로 따라가는 체크리스트.
> 코드 쪽 준비(업로드 키 서명 설정, 아이콘, 구독 연동)는 이미 끝나 있다 — 이 문서는 **맥 터미널 + Play Console에서 하는 일**만 다룬다.
> 막히는 화면은 스크린샷 찍어서 Claude에게 보여주면 된다 (App Store 때와 동일한 진행 방식).

## 0. 시작 전 결정: 개인 계정 vs 사업자(조직) 계정

| | 개인 계정 | 사업자(조직) 계정 |
|---|---|---|
| 등록비 | $25 일회성 | $25 일회성 |
| 필요 서류 | 신분증 본인 인증 | **D-U-N-S 번호** + 사업자 정보 |
| 프로덕션 출시 요건 | ⚠️ **비공개 테스트: 테스터 12명 이상 × 14일 연속** 통과 필수 | 요건 없음 — 바로 출시 가능 |
| 준비 기간 | 즉시 가입 가능 | D-U-N-S 발급에 수일~수주 |

- 2023년 11월 이후 만든 **개인** 계정에만 12명×14일 요건이 붙는다.
- **빨리 내고 싶다** → 개인 계정 + 지인 12명 모집.
- **테스터 못 모은다 / 셀퍼 명의로 내고 싶다** → 사업자 계정 + D-U-N-S 신청(무료, https://www.dnb.com 또는 던앤브래드스트리트 코리아).
- 어느 쪽이든 아래 절차는 동일하고, 5단계의 "비공개 테스트 14일"만 개인 계정에서 추가된다.

## 1. Play Console 개발자 계정 만들기

1. https://play.google.com/console → **윤세현 Google 계정**으로 가입 (애플과 명의 통일).
2. 계정 유형 선택 (0단계에서 정한 것) → 등록비 **$25** 결제 (해외결제 되는 카드).
3. 본인 인증: 신분증 촬영 + 전화·이메일 인증. 승인까지 며칠 걸릴 수 있다.
   → **대기하는 동안 2·3단계를 먼저 해두면 시간 낭비 없음.**

## 2. 업로드 키스토어 만들기 (맥, 최초 1회만)

Play는 "업로드 키"로 서명된 앱 번들만 받는다. 키를 만들고, 빌드가 그 키로 서명되게 연결한다.

### 2-1. 키 생성

터미널에 붙여넣기 (`비밀번호`는 직접 정한다 — 두 번 물어봄):

```bash
cd ~/routine/android
keytool -genkeypair -v -keystore upload-keystore.jks -alias upload \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -dname "CN=Sehyun Youn, O=Selfer, C=KR"
```

→ `~/routine/android/upload-keystore.jks` 생성됨.

### 2-2. 비밀번호 연결 파일

`~/routine/android/keystore.properties` 파일을 새로 만들어 아래처럼 채운다:

```
storeFile=upload-keystore.jks
storePassword=정한비밀번호
keyAlias=upload
keyPassword=정한비밀번호
```

### 2-3. 절대 규칙

- ⚠️ **`upload-keystore.jks`와 비밀번호를 잃어버리면 앱 업데이트를 영영 못 올린다.**
  파일은 클라우드 등 안전한 곳에 백업, 비밀번호는 별도 기록.
- 두 파일 모두 `.gitignore` 처리되어 있어 깃에는 올라가지 않는다 (올리면 안 됨).

## 3. 출시용 앱 번들(AAB) 빌드

```bash
cd ~/routine && git pull
cd android && ./gradlew bundleRelease
```

- 결과물: **`android/app/build/outputs/bundle/release/app-release.aab`** ← Play에 올릴 파일.
- 버전 올릴 때는 `android/app/build.gradle.kts`의 `versionCode`(정수, 업로드마다 +1)와 `versionName`(표시용, 예: 1.0.1)을 수정 — Claude에게 "버전 올려줘"라고 하면 됨.

> 참고: Play가 자체 "앱 서명 키"로 최종 서명을 다시 한다(Play App Signing).
> 그래서 **6-2의 SHA-1 등록**을 빼먹으면 출시판에서 Google 로그인이 죽는다 — 잊지 말 것.

## 4. Play Console에서 앱 만들기 + 필수 양식

**앱 만들기** → 이름 `앵그리모티`, 기본 언어 `한국어`, 앱, 무료.

대시보드의 "앱 설정" 체크리스트를 위에서부터 채운다. 답안지:

| 항목 | 답 |
|---|---|
| 개인정보처리방침 URL | 노션 링크 (iOS 심사 때 것 그대로) |
| 앱 액세스 권한 | 전체 기능 사용 가능 (로그인 필요 시: 게스트 모드로 모든 핵심 기능 접근 가능이라고 서술) |
| 광고 | 광고 없음 |
| 콘텐츠 등급 설문 | 전부 "아니오" → 전연령 등급 |
| 타겟층 | 만 18세 이상 (아동 대상 아님 — 심사 단순화) |
| 뉴스 앱 | 아니오 |
| 데이터 보안 | 아래 4-1 참고 |
| 정부 앱 | 아니오 |

### 4-1. 데이터 보안 양식

- **수집함**: 이메일 주소, 이름 — 목적 "계정 관리" / 선택사항(게스트 모드 있음) / 전송 중 암호화 / 사용자가 삭제 요청 가능(앱 내 계정 탈퇴).
- **촬영 영상·사진**: **수집 안 함** — 기기에만 저장되고 서버로 전송되지 않음. 양식에서 "수집"의 정의가 '기기 밖으로 전송'이므로 당당하게 수집 안 함.
- 위치·연락처·검색 기록 등 나머지: 전부 수집 안 함.

### 4-2. 민감 권한 선언 (앱 콘텐츠 메뉴)

| 권한 | 선언 내용 |
|---|---|
| `USE_EXACT_ALARM` | "앱의 핵심 기능이 알람" 항목 선택. 앵그리모티는 예약 시각 정밀 알람이 본질 기능이라 정당. |
| 포그라운드 서비스 (camera) | 사용 이유: "사용자가 예약한 활동 시간 동안 타임랩스 촬영을 유지하기 위함". **기능 시연 영상 요구됨** — 알람 울림 → 촬영 시작 → 세션 화면 흐름을 폰 화면녹화로 30초~1분 찍어 유튜브 '일부 공개(unlisted)'로 올리고 링크 제출. |
| `READ_PHONE_STATE` | 사용 이유: "촬영 중 전화 수신 시 벌점 없이 자동 일시정지하기 위한 통화 상태 감지". 반려되면 이 권한을 빼는 코드 우회안이 있으니 Claude에게 알릴 것. |

## 5. 내부 테스트 → 비공개 테스트

### 5-1. 내부 테스트 (심사 없음, 즉시)

1. 테스트 → **내부 테스트** → 새 버전 만들기 → 3단계의 `app-release.aab` 업로드.
2. 테스터 목록에 본인 Gmail 추가 → 참여 링크로 폰에서 설치 → 앱 정상 동작 확인.

### 5-2. 비공개 테스트 (개인 계정만 필수)

1. 테스트 → **비공개 테스트** → 트랙 만들기 → 같은 AAB 승격.
2. 테스터 12명 이상 이메일 등록, 각자 참여 링크로 **옵트인 + 설치**해야 카운트된다.
3. **14일 연속** 유지 후 프로덕션 신청 자격이 생김.
   → 이 14일 동안 6·7단계를 병행하면 낭비 없음.

## 6. 수익 창출 (구독) 설정

### 6-1. 판매자 프로필 + 지급 계좌

수익 창출 설정 → 판매자 프로필 작성 + **지급용 은행 계좌** 등록 (애플 때처럼 검증 대기 있음).
한국 판매자 정보에 **사업자등록번호** 입력 (셀퍼 것 재사용).

### 6-2. Firebase에 Play 서명 키 SHA-1 등록 (Google 로그인 필수)

1. Play Console → 설정 → **앱 무결성 → 앱 서명** → "앱 서명 키 인증서"의 **SHA-1** 복사.
2. Firebase 콘솔 → 프로젝트 설정 → Android 앱(`com.selfer.angrymoti`) → **디지털 지문 추가** → 붙여넣기.
3. 업로드 키 SHA-1도 함께 등록: `cd ~/routine/android && ./gradlew signingReport`의 SHA1 값.
   → 이 단계를 빼먹으면 **스토어에서 받은 앱에서만 Google 로그인이 실패**하는 미스터리 버그가 된다.

### 6-3. 구독 상품 만들기

수익 창출 → 구독 → 만들기:

| 항목 | 값 |
|---|---|
| 상품 ID | **`com.timelock.pro.monthly`** ← 코드와 정확히 일치해야 페이월이 뜸 (변경 불가) |
| 이름 | 앵그리모티 멤버십 |
| 기본 요금제 | 월간 자동 갱신, **₩4,400** (iOS와 통일) |

### 6-4. 결제 테스트

설정 → 라이선스 테스트 → 본인 Gmail 등록 → 내부 테스트 빌드에서 구독하면 **테스트 카드**로 무과금 결제 가능. 페이월 → 결제 → 멤버십 활성(워터마크 제거 토글 등) 확인.

## 8. 프로덕션 출시

1. (개인 계정) 비공개 테스트 14일 충족 → 대시보드에 "프로덕션 신청" 절차 등장 → 설문 작성.
2. 프로덕션 → 새 버전 만들기 → AAB 승격 → **출시 검토 후 제출**.
3. 심사는 보통 몇 시간~수일. 반려되면 사유 메시지를 Claude에게 그대로 붙여넣기.
4. 승인 후 출시 방식: 관리형 게시(수동 출시)로 해두면 애플의 '수동 출시'와 동일하게 원하는 시점에 공개 가능.

## 9. 출시 후 업데이트 루틴

```bash
cd ~/routine && git pull          # 코드 최신화
# build.gradle.kts에서 versionCode +1, versionName 수정 (Claude에게 요청)
cd android && ./gradlew bundleRelease
# Play Console → 프로덕션 → 새 버전 → AAB 업로드 → 제출
```

## 부록. 자주 막히는 지점

- **"이 App Bundle은 이미 사용된 versionCode..."** → `versionCode`를 안 올리고 재업로드한 것. +1 후 재빌드.
- **출시판에서만 Google 로그인 실패** → 6-2의 SHA-1 등록 누락.
- **페이월이 "불러오는 중"에서 안 넘어감** → 구독 상품 ID 불일치 또는 상품이 '활성' 상태가 아님. 내부 테스트 트랙에 앱이 올라가 있어야 상품 조회가 된다.
- **키스토어 비밀번호 분실** → 업로드 키는 Play 고객센터를 통해 재설정 가능(Play App Signing 사용 중이므로 복구 여지 있음)이지만 절차가 길다. 백업이 최선.
