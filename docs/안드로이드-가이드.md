# 앵그리모티 — 안드로이드 버전 가이드

> 코드 위치: 저장소 `android/` 폴더 (Kotlin + Jetpack Compose, minSdk 26 / target 34).
> **정책은 iOS와 1:1 동일** — 점수·슬롯·자리비움·긴급 예산 전부 README 명세 그대로 (`models/Models.kt`).

## 1. 처음 여는 법 (Android Studio)

1. Android Studio (Koala 이상) → **Open** → 저장소의 `android` 폴더 선택
2. Gradle Sync가 자동으로 돌면서 의존성 다운로드 (수 분)
3. 실기기 연결 (개발자 옵션 → USB 디버깅) → ▶ Run

> 시뮬레이터(에뮬레이터)는 전면 카메라가 가짜 영상이라 촬영 흐름 확인용으로만 쓰고, 실제 테스트는 실기기로.

## 2. Firebase 연결 (iOS와 계정·점수 공유)

같은 프로젝트(`timelock-eba85`)에 Android 앱만 추가하면 iOS와 데이터가 공유된다.

1. Firebase 콘솔 → 프로젝트 설정 → **앱 추가 → Android**
2. 패키지 이름: **`com.singlemarks.angrymoti`**
3. **SHA-1 등록** (Google 로그인·인증에 필요):
   - Android Studio 터미널에서 `./gradlew signingReport` → `SHA1:` 값 복사해서 입력
4. **`google-services.json` 다운로드 → `android/app/` 폴더에 넣기**
5. 다시 빌드하면 Firebase 활성화 (파일이 없으면 iOS처럼 '기기 내 게스트' 모드로만 동작)

> ⚠️ 저장소가 공개이므로, Google Cloud 콘솔에서 이 Android API 키도
> "Android 앱 제한(패키지명+SHA-1)"을 걸어둘 것 (iOS 키에 한 것과 동일).

## 3. Google Play Console 배포 절차

1. **개발자 계정 등록** — play.google.com/console, $25 일회성
2. ⚠️ **신규 개인 계정 요건**: 정식 출시 전에 **비공개 테스트 (테스터 12명+, 14일 연속)** 필수
3. 앱 만들기 → 패키지 `com.singlemarks.angrymoti`
4. **서명 키 생성** (최초 1회):
   - Android Studio → Build → Generate Signed App Bundle → 키스토어 새로 만들기
   - 키스토어 파일·비밀번호는 절대 분실 금지 (저장소에 커밋하지 말 것)
5. Build → **Generate Signed App Bundle (.aab)** → Play Console 비공개 테스트 트랙에 업로드
6. **구독 상품 등록**: 수익화 → 구독 → 상품 ID **`com.timelock.pro.monthly`** (코드와 일치 필수),
   기본 요금제: 월간 ₩4,900. 등록 전까지 페이월은 "불러오는 중" 표시.
7. 스토어 등록정보: 이름 앵그리모티 / 아이콘·스크린샷 / 개인정보처리방침 URL(노션) — iOS와 동일 값
8. **데이터 보안 양식**: 수집 = 이메일·이름(계정 기능, 필수 아님-게스트 가능), 영상은 기기에만 저장·전송 없음
9. 한국 판매자 정보: 사업자등록번호 (애플 때 만든 것 재사용)

## 4. iOS ↔ Android 기술 대응표

| 기능 | iOS | Android |
|---|---|---|
| UI | SwiftUI | Jetpack Compose |
| 저장 | SwiftData | Room |
| 알람 | UserNotifications | AlarmManager(setAlarmClock) + 풀스크린 알림 |
| 카메라 | AVFoundation | CameraX + MediaCodec(H.264) |
| 부재 감지 | Vision | ML Kit Face Detection (온디바이스) |
| 통화 감지 | CXCallObserver | TelephonyCallback (READ_PHONE_STATE 선택 권한) |
| 구독 | StoreKit 2 | Google Play Billing 7 |
| 세션 보호 | 자체 | **포그라운드 서비스 + WakeLock** (OEM 킬 방어) |

## 5. 안드로이드 특유 주의점 (테스트 시 확인)

- **정확한 알람**: 설정에서 "알람 및 리마인더" 권한이 켜져 있어야 함 (안 켜져 있으면 알람이 못 울림 — 온보딩 이후 시스템이 자동 허용하는 기종이 대부분이나 일부 기종은 수동)
- **삼성 배터리 최적화**: 설정 → 배터리 → 앵그리모티를 "제한 없음"으로 — 장시간 세션 테스트 전 권장
- **타임랩스 인코딩**: 소프트웨어 YUV 방식이라 모든 기기에서 동작하도록 설계했지만, 첫 실기기 테스트에서 결과 영상 재생을 꼭 확인할 것
- **통화 일시정지**: READ_PHONE_STATE 권한 거부 시 통화가 이탈(자동 긴급 중단)로 처리됨 — 정책상 허용 범위

## 6. 남은 작업 (알려진 한계)

- Google 로그인 버튼: Firebase 연결(2번) + SHA-1 등록 후 활성화 예정 (현재 이메일/게스트 동작)
- 세션 화면 가로 모드 레이아웃: v1은 세로 중심 (가로 선택 시 영상은 가로로 저장됨)
- 랭킹게임: iOS와 동일하게 "준비 중"
